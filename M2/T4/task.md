## 1. Выбор предобученной модели

* Выберите понравившуюся вам предобученную языковую модель на **Hugging Face Model Hub** (по размеру — от **0.5B** параметров)
* Загрузите выбранную модель и токенизатор в **4-bit режиме**. Настройте **QLoRA**. Используйте `chat_template` токенизатора для форматирования диалогов.

## 2. Подготовка валидации

* **Harmful**: выберите **100 пар** из `Anthropic/hh-rlhf` (обратите внимание — строки датасета представляют собой плоские диалоги, их необходимо привести в чат-формат, понятный instruct-модели)
* Зафиксируйте **seed** и одинаковые параметры декодирования для всех сравнений (например: `temperature = 0.8` для helpful, `1.0` для harmful; `top_p = 0.95`)

## 3. Бейзлайн (до DPO)

* Сгенерируйте ответы модели для валидационного **harmful-набора**
* Посчитайте:
    * **mean toxicity** на harmful (например, через `detoxify`)
    * **refusal rate** на harmful:
        * простой вариант — текстовая эвристика (наличие в ответе: "I can't", "I cannot", "sorry", "won't", "unable", "I haven't", "I'm not", "I'm sorry" и т.п.)
        * более сложный вариант — через **LLM as Judge**

## 4. DPO + QLoRA

* Соберите **train-датасет 2000–6000 пар** из `Anthropic/hh-rlhf`
* Обучите модель с **DPO**, подберите оптимальные параметры
* Сохраните **LoRA-веса**

## 5. Оценка после DPO

* Повторите генерацию на валидационном наборе с теми же параметрами декодирования
* Посчитайте:
    * **mean toxicity** (`detoxify`) и дельту относительно бейзлайна
    * **refusal rate** и дельту

## 6. Приём без RL — constitutional rewrite
* Задайте мини-конституцию (**3–5 правил**), например:
    * не давать вредные инструкции
    * избегать токсичной лексики
    * если unsafe — вежливый отказ + безопасные альтернативы
* Оригинальной моделью **до DPO** для harmful-ответов выполните два шага:
    * **critique** — выявление проблем
    * **rewrite** — переписывание под правила
* Посчитайте **mean toxicity** и **refusal rate** для ответов *до переписывания* и *после*, сделайте выводы
